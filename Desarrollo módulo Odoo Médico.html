🔹 2. Instalación del Entorno en Local

Si aún no tienes Odoo 18 Enterprise, sigue estos pasos:

🔸 1. Instalar dependencias

bash
sudo apt update && sudo apt upgrade -y
sudo apt install python3 python3-pip python3-venv git postgresql -y

🔸 2. Crear una base de datos en PostgreSQL

bash
sudo -i -u postgres
psql
CREATE DATABASE odoo18;
CREATE USER odoo WITH ENCRYPTED PASSWORD 'odoo';
ALTER ROLE odoo SET client_encoding TO 'utf8';
ALTER ROLE odoo SET default_transaction_isolation TO 'read committed';
ALTER ROLE odoo SET timezone TO 'UTC';
GRANT ALL PRIVILEGES ON DATABASE odoo18 TO odoo;
\q
exit

🔸 3. Clonar Odoo Enterprise (si tienes acceso)

bash
git clone https://github.com/odoo/enterprise.git -b 18.0 odoo-enterprise

🔸 4. Crear un entorno virtual y activar Odoo

bash
python3 -m venv odoo-venv
source odoo-venv/bin/activate
pip install -r odoo/requirements.txt

🔸 5. Ejecutar el servidor Odoo

bash
python odoo/odoo-bin --addons-path=odoo/addons,odoo-enterprise --db_host=localhost --db_user=odoo --db_password=odoo --db_name=odoo18

Después de esto, puedes abrir http://localhost:8069 y acceder a Odoo.

from odoo import models, fields, api

class CitaMedica(models.Model):
    _name = 'gestion_citas.cita'
    _description = 'Cita Médica'

    paciente_id = fields.Many2one('gestion_citas.paciente', string="Paciente", required=True)
    fecha_cita = fields.Datetime(string="Fecha y Hora", required=True)
    medico = fields.Char(string="Médico")
    estado = fields.Selection([
        ('pendiente', 'Pendiente'),
        ('confirmada', 'Confirmada'),
        ('cancelada', 'Cancelada')
    ], default='pendiente', string="Estado")

Si te sale un error al hacer git push origin main, probablemente sea porque:

  1. No has autenticado Git con tu cuenta de GitHub.
  2. No tienes permisos en el repositorio.
  3. La rama main no existe en GitHub.

🔹 Solución Paso a Paso:

1️⃣ Verifica la conexión con GitHub:

bash

Si no ves la URL de tu repo, agrega el remoto:

2️⃣ Autentica con GitHub:
Si GitHub te pide autenticación, usa un Personal Access Token (PAT) en lugar de la contraseña.

  • Genera un token en GitHub Tokens.
  • Usa el token en lugar de la contraseña cuando Git lo solicite.

3️⃣ Sube la rama correctamente:

bash
git push --set-upstream origin main

Si el error persiste, dime qué mensaje exacto te muestra. 🚀
